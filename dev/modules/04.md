# Module 5: Transfer Functions and Linearized Process Models

## Overview

Once Laplace transforms are available, we can express a linear dynamic model in
a form that is even more useful for control design: the transfer function. A
transfer function relates an input to an output in the Laplace domain and
provides a compact summary of dynamic behavior.

This representation is powerful because it shifts attention from the internal
details of every derivative term to the input-output behavior that matters for
analysis and control. The gain, dynamic order, poles, and zeros of the model
become visible directly. Processes can be connected in series through simple
multiplication, and standard response patterns become easier to recognize.

In practice, transfer functions are rarely derived from nothing. They usually
begin with a differential-equation model obtained from balances or from a
simplified physical argument. That model is then written in deviation variables,
transformed into the Laplace domain, and rearranged into a ratio of output to
input.

This module develops that workflow. We begin with the definition of a transfer
function, show why deviation variables matter, derive transfer functions from
simple process models, examine their main properties, and then discuss how
nonlinear models are linearized so that transfer-function methods can be used.

## From Differential Equations to Input-Output Models

A time-domain process model often takes the form

```text
dynamic accumulation terms = input effects - output effects
```

For a single-input, single-output system, we may write an ordinary differential
equation relating an input `u(t)` and an output `y(t)`. After taking Laplace
transforms and assuming suitable initial conditions, that same model can be
expressed as

```text
Y(s) = G(s) U(s)
```

where `G(s)` is the transfer function.

By definition,

```text
G(s) = Y(s) / U(s)
```

This ratio is meaningful only when the model is linear and the initial
conditions are handled consistently, usually by assuming zero initial deviation.

## Why Deviation Variables Are Used

Transfer functions are most naturally written in deviation variables rather than
absolute physical variables.

If `y` is the nominal steady-state value of the output and `u` is the nominal
steady-state value of the input, then the deviation variables are

```text
y' = y - y_bar
u' = u - u_bar
```

This change of variables matters for three reasons.

First, it aligns the model with the way process disturbances and control actions
are usually described: as changes away from a nominal operating point.

Second, it removes constant steady-state offsets from the equations and leaves a
cleaner dynamic model.

Third, it supports the standard transfer-function assumption of zero initial
conditions in deviation form.

## Worked Example 1: Liquid Level in a Storage Tank

Consider a liquid storage tank with cross-sectional area `A`, inlet flow `q_i`,
liquid level `h`, and outlet flow proportional to the level:

```text
q_o = h / R
```

The dynamic balance is

```text
A dh/dt = q_i - q_o
```

Substituting the outlet relation gives

```text
A dh/dt = q_i - h / R
```

### Step 1: Write the model in deviation variables

At the nominal steady state,

```text
0 = q_i_bar - h_bar / R
```

Subtracting the steady-state equation from the dynamic equation gives

```text
A dh'/dt = q_i' - h' / R
```

### Step 2: Transform the equation

Assuming zero initial deviation,

```text
A s H'(s) = Q_i'(s) - H'(s) / R
```

Rearranging,

```text
(A s + 1/R) H'(s) = Q_i'(s)
```

so

```text
H'(s) / Q_i'(s) = R / (A R s + 1)
```

### Step 3: Interpret the result

This is a first-order transfer function of the standard form

```text
G(s) = K / (tau s + 1)
```

with

```text
K = R
tau = A R
```

The gain tells us how much the level changes at steady state for a sustained
change in inlet flow. The time constant tells us how quickly the tank responds.

### What this example teaches

A physically transparent balance equation becomes an equally transparent
transfer-function model. The gain and time constant emerge directly from process
parameters.

## Standard First-Order Transfer Function

Many process units can be approximated by

```text
G(s) = K / (tau s + 1)
```

This form appears for tanks, thermal systems, concentration dynamics, and many
actuated process elements.

Its importance is hard to overstate. It tells us that a large class of physical
processes can be understood with only two parameters:

- `K`, the steady-state gain
- `tau`, the time constant

In time-domain language, `K` tells us how large the final response will be, and
`tau` tells us how fast the process moves.

## Transfer Functions as Building Blocks

One of the main advantages of transfer functions is that they combine easily.
If one subsystem feeds directly into another, the overall transfer function is
the product of the two individual transfer functions.

Suppose

```text
G1(s) = Y1(s) / U(s)
G2(s) = Y(s) / Y1(s)
```

Then the overall relation is

```text
G(s) = Y(s) / U(s) = G2(s) G1(s)
```

This multiplicative property is extremely useful in process control because real
systems are often composed of units connected in series: sensor, process,
actuator, valve, transmitter, and controller all contribute dynamic effects.

## Worked Example 2: Two Surge Tanks in Series

Consider two first-order tanks connected in series. Each tank behaves like a
first-order element:

```text
G1(s) = K1 / (tau1 s + 1)
G2(s) = K2 / (tau2 s + 1)
```

If the output of Tank 1 becomes the input to Tank 2, then the overall transfer
function is

```text
G(s) = G2(s) G1(s)
     = K1 K2 / [(tau1 s + 1)(tau2 s + 1)]
```

This is a second-order transfer function.

### Physical interpretation

The overall process now contains two dynamic lags. The response is slower and
its shape is richer than that of a single first-order system. Even when each
individual unit is simple, the combined system can become significantly more
complicated.

### What this example teaches

Transfer functions allow us to build larger models from smaller pieces while
preserving physical meaning. This makes them especially useful for flowsheets
and control-loop analysis.

## Properties of Transfer Functions

Transfer functions are useful not only because they are compact, but also
because they reveal important dynamic features directly.

### Steady-State Gain

The gain tells us how strongly the output responds to a sustained input change.
For many stable systems, it can be interpreted from the low-frequency or
steady-state behavior of the model.

In practical terms, gain answers the question:

```text
If I hold the input at a new value, how far will the output eventually move?
```

### Order

The order of a transfer function is tied to the order of the differential
equation from which it came. A first-order system has one dynamic storage mode,
a second-order system has two, and so on.

Higher-order systems can often show more complex behavior, including multiple
time scales, overshoot, oscillation, or interactions between fast and slow
dynamics.

### Poles

The poles of a transfer function are the values of `s` that make the
denominator zero.

Poles matter because they determine the natural dynamic modes of the process.
For stable first-order systems, poles lie in the left half of the complex plane
and correspond to decaying exponentials in time.

Although detailed pole analysis is usually developed later in a course, the
basic intuition begins here: denominator structure controls dynamic behavior.

### Zeros

Zeros are the values of `s` that make the numerator zero.

Zeros do not usually determine whether a process is stable, but they strongly
affect how the output responds to an input. They can speed up or reshape the
transient response and sometimes create counterintuitive behavior.

### Relative Degree and Properness

In most physical process models, the order of the denominator is at least as
large as the order of the numerator. This corresponds to the idea that physical
systems store mass, energy, or momentum and therefore cannot respond with
infinite speed.

This observation is one way of connecting the mathematics of transfer functions
back to physical realism.

## Block-Diagram Thinking

Transfer functions are often arranged visually in block diagrams. A block
diagram does not show physical geometry directly. Instead, it shows the flow of
information or causal influence from one variable to another.

This is useful because control systems are organized by signal relationships:

- the process receives an input
- the process generates an output
- the sensor measures the output
- the controller computes a new action
- the final control element changes the manipulated variable

Transfer functions fit this signal-flow view perfectly. Each block represents
an input-output relation, and complicated loops can be analyzed by combining the
blocks systematically.

## Why Nonlinear Models Must Often Be Linearized

Many real processes are nonlinear. Reaction rates depend exponentially on
temperature, valve relations may be nonlinear, pH systems are strongly
nonlinear, and distillation compositions often respond asymmetrically to upward
and downward changes.

Classical control methods, however, are mostly built on linear models. For that
reason, nonlinear process models are often linearized around a nominal
steady-state operating point.

The idea is not that the nonlinearity disappears. The idea is that near a
selected operating condition, the nonlinear model can be approximated well
enough by a linear one to support analysis and controller design.

## Linearization by Taylor Series Expansion

Suppose a nonlinear single-input model has the form

```text
dy/dt = f(y, u)
```

We choose a nominal steady state `(y_bar, u_bar)` and expand `f` in a first-order
Taylor series around that point:

```text
f(y, u) approximately equals
f(y_bar, u_bar)
+ (df/dy)_s (y - y_bar)
+ (df/du)_s (u - u_bar)
```

At steady state, `f(y_bar, u_bar) = 0`, so the linearized deviation-variable
model becomes

```text
dy'/dt = a y' + b u'
```

where

```text
a = (df/dy)_s
b = (df/du)_s
```

This is now a linear differential equation, which means Laplace transforms and
transfer functions can be used.

## Worked Example 3: Linearizing a Nonlinear Process

Suppose a nonlinear process is modeled by

```text
dy/dt = -y^2 + 3 u
```

Let the nominal operating point be `(y_bar, u_bar)`. The first-order Taylor
approximation gives

```text
dy'/dt = (-2 y_bar) y' + 3 u'
```

Taking Laplace transforms with zero initial deviation,

```text
s Y'(s) = (-2 y_bar) Y'(s) + 3 U'(s)
```

so the transfer function is

```text
Y'(s) / U'(s) = 3 / (s + 2 y_bar)
```

### Physical interpretation

The local dynamics depend on the nominal operating point. This is one of the
most important lessons in nonlinear process control: a process may behave
differently at different steady states, even though the equipment itself has not
changed.

### What this example teaches

Linearization is not just a mathematical trick. It is a way of obtaining a
locally valid control model from a more realistic nonlinear description.

## Limits of Linearization

A linearized model is accurate only near the operating point where it was
derived. If disturbances become large or the process moves far from the nominal
condition, the approximation may become poor.

This limitation should always be kept in mind. A transfer function is often
extremely useful, but it is still an approximation of a physical process, not
the process itself.

For many continuous process-control problems, this is acceptable because the
system spends long periods near a nominal operating condition and only small or
moderate deviations are expected during normal operation.

## Why These Models Matter for Control Design

Transfer functions and linearized models are central because they support nearly
every major activity in classical control:

- predicting step responses
- analyzing disturbance rejection
- identifying dominant time constants
- combining subsystems in series or feedback
- evaluating the effect of poles and zeros
- designing and tuning controllers

They are therefore not a mathematical side topic. They are the main language
used to connect process physics to controller behavior.

## Key Takeaways

A transfer function is the Laplace-domain ratio of output to input for a linear
system under consistent initial-condition assumptions. It provides a compact
input-output description of process dynamics.

Deviation variables are used because they center the model around a nominal
steady state and simplify the mathematics. Many important process units reduce
to first-order transfer functions characterized by a steady-state gain and a
time constant.

Transfer functions combine naturally in series, making them ideal building
blocks for larger systems and block-diagram analysis. Their poles, zeros, and
order provide direct insight into dynamic behavior.

Because many real processes are nonlinear, linearization around a steady-state
operating point is often required before transfer-function methods can be
applied. This local approximation is one of the most important modelling ideas
in classical process control.
